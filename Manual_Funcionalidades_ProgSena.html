1. Sistema de Notificaciones

1.1 Notificaciones Bidireccionales

Descripción: Sistema completo de notificaciones entre coordinadores e instructores.

Características:
  • Notificaciones Coordinador → Instructor
  • Notificaciones Instructor → Coordinador (cambios de perfil)
  • Actualización automática cada 30 segundos
  • Badge con contador de notificaciones no leídas
  • Tipos: General, Asignación, Cambio de Horario, Recordatorio, Urgente
Tablas de Base de Datos:
CREATE TABLE notificaciones_instructor ( id INT AUTO_INCREMENT PRIMARY KEY, instructor_id INT NOT NULL, coordinador_id INT NOT NULL, tipo VARCHAR(50) DEFAULT 'general', titulo VARCHAR(255) NOT NULL, mensaje TEXT NOT NULL, leida TINYINT(1) DEFAULT 0, fecha_creacion TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); CREATE TABLE notificaciones_coordinador ( id INT AUTO_INCREMENT PRIMARY KEY, instructor_id INT NOT NULL, coordinador_id INT NOT NULL, instructor_nombre VARCHAR(200), tipo VARCHAR(50) DEFAULT 'general', titulo VARCHAR(255) NOT NULL, mensaje TEXT NOT NULL, leida TINYINT(1) DEFAULT 0, fecha_creacion TIMESTAMP DEFAULT CURRENT_TIMESTAMP );
API Endpoints:
POST /api/notificaciones.php?action=enviar_notificacion
Envía notificación del coordinador al instructor
POST /api/notificaciones.php?action=enviar_notificacion_coordinador
Envía notificación del instructor al coordinador
GET /api/notificaciones.php?action=listar_notificaciones_coordinador
Lista notificaciones del coordinador

2. Gestión de Asignaciones

2.1 Crear Asignaciones

Descripción: Sistema completo para crear y gestionar asignaciones de instructores a fichas.

Campos del Formulario:
Campo Tipo Descripción
Ambiente Select Aula, laboratorio o taller
Competencia Select Experiencia formativa
Instructor Select Instructor asignado
Ficha Select Código de ficha
Días de la semana Checkboxes Lunes a Sábado
Rango de fechas Date Fecha inicio y fin
Rango de horas Select Intervalos de 15 minutos
Tabla de Base de Datos:
CREATE TABLE asignaciones ( asignacion_id INT AUTO_INCREMENT PRIMARY KEY, ficha_id INT NOT NULL, instructor_id INT NOT NULL, experiencia_id INT NOT NULL, ambiente_id INT DEFAULT NULL, fecha_inicio DATE NOT NULL, fecha_fin DATE NOT NULL, hora_inicio TIME DEFAULT NULL, hora_fin TIME DEFAULT NULL, dias_semana VARCHAR(50) DEFAULT NULL, estado ENUM('Programada', 'En Curso', 'Finalizada', 'Cancelada'), observaciones TEXT DEFAULT NULL, fecha_creacion TIMESTAMP DEFAULT CURRENT_TIMESTAMP );
API Endpoints:
POST /api/asignaciones.php
action=crear - Crea nueva asignación
GET /api/asignaciones.php?action=listar
Lista todas las asignaciones con información completa
POST /api/asignaciones.php
action=actualizar - Actualiza asignación existente
POST /api/asignaciones.php
action=eliminar - Elimina asignación

3. Sistema de Calendario

3.1 FullCalendar Integration

Descripción: Calendario interactivo con gestión de eventos y asignaciones.

Características:
  • Vistas: Mes, Semana, Día, Lista
  • Idioma: Español
  • Horario: 06:00 AM - 10:00 PM
  • Eventos agrupados por asignación (groupId)
  • Eliminación en grupo de eventos relacionados
  • Drag & Drop para mover eventos
  • Colores personalizados por tipo
Tabla de Base de Datos:
CREATE TABLE events ( id INT AUTO_INCREMENT PRIMARY KEY, title VARCHAR(255) NOT NULL, start_date DATETIME NOT NULL, end_date DATETIME DEFAULT NULL, user_id INT DEFAULT NULL, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP );
API Endpoints:
GET /api/calendar_events.php
Lista todos los eventos del calendario
POST /api/calendar_events.php
Crea nuevo evento en el calendario
DELETE /api/calendar_events.php?id={id}
Elimina evento del calendario

4. Edición de Perfil del Instructor

4.1 Actualización de Perfil

Descripción: Sistema completo para que los instructores actualicen su información personal.

Campos Editables:
  • Nombre y Apellido
  • Email
  • Teléfono
  • Documento
  • Especialidad
  • Biografía
  • Cambio de contraseña (opcional)
Flujo de Actualización:
  1. Instructor edita su perfil
  2. Sistema guarda cambios en la base de datos
  3. Se envía notificación automática al coordinador
  4. Coordinador revisa y aprueba cambios
  5. Historial de cambios se registra
Validaciones:
  • Campos obligatorios marcados con *
  • Validación de formato de email
  • Contraseñas deben coincidir
  • Actualización en tiempo real sin recargar página

5. Sistema de Contexto de Rol

5.1 Múltiples Pestañas con Diferentes Roles

Descripción: Sistema que permite trabajar con múltiples roles en pestañas separadas sin conflictos.

Arquitectura:
  • Frontend: sessionStorage para contexto por pestaña
  • Backend: $_SESSION['contextos'] separados por rol
  • ID único por pestaña (tabId)
  • Registro de pestañas activas en localStorage
  • Keep-alive automático cada 30 segundos
  • Limpieza automática al cerrar pestaña
Archivos Implementados:
  • role_context.php: Sistema backend de contexto
  • assets/js/role_context.js: Sistema frontend
Funciones Principales:
// PHP inicializarContextoRol($rol) guardarEnContexto($rol, $clave, $valor) obtenerDeContexto($rol, $clave) verificarAccesoRol($rolRequerido) // JavaScript initRoleContext(rol) getOrCreateTabId() markTabActive() keepAlive() cleanup()

6. Triggers y Validaciones de Base de Datos

6.1 Triggers Implementados

1. Validación de Carga Horaria:
CREATE TRIGGER before_asignacion_insert_check BEFORE INSERT ON asignaciones FOR EACH ROW BEGIN IF NEW.horas_semanales > 20 THEN SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = 'Error: No se puede asignar más de 20 horas'; END IF; END;
2. Validación de Conflicto de Instructor:
CREATE TRIGGER before_asignacion_instructor_check BEFORE INSERT ON asignaciones FOR EACH ROW BEGIN -- Verifica que el instructor no tenga otra asignación -- en el mismo horario y días END;
3. Validación de Conflicto de Ambiente:
CREATE TRIGGER before_asignacion_ambiente_check BEFORE INSERT ON asignaciones FOR EACH ROW BEGIN -- Verifica que el ambiente no esté ocupado -- en el mismo horario y días END;
4. Auditoría de Cambios:
CREATE TRIGGER after_asignacion_update_audit AFTER UPDATE ON asignaciones FOR EACH ROW BEGIN INSERT INTO auditoria_asignaciones (asignacion_id, usuario_que_creo, fecha_registro, detalles) VALUES (NEW.asignacion_id, 'SISTEMA', NOW(), CONCAT('Cambio de ', OLD.estado, ' a ', NEW.estado)); END;

7. Optimizaciones del Sistema

7.1 Configuración de Sesiones

Archivo: session_config.php

Configuraciones Clave:
ini_set('session.use_strict_mode', 1); ini_set('session.cookie_httponly', 1); ini_set('session.use_only_cookies', 1); ini_set('session.cookie_samesite', 'Lax'); ini_set('session.gc_maxlifetime', 7200); // 2 horas ini_set('session.lazy_write', 1); // Clave para múltiples pestañas
Beneficios:
  • Múltiples pestañas sin conflictos
  • Mayor seguridad
  • Mejor rendimiento
  • Sesiones estables

7.2 Conexión a Base de Datos

Archivo: connection.php

Optimizaciones:
$pdo_options[PDO::ATTR_PERSISTENT] = false; // No persistente $pdo_options[PDO::ATTR_TIMEOUT] = 5; // Timeout 5 segundos $pdo_options[PDO::ATTR_AUTOCOMMIT] = 1; // Autocommit $pdo_options[PDO::MYSQL_ATTR_INIT_COMMAND] = "SET NAMES utf8mb4";
