2. Consultas de Autenticación y Usuarios

2.1 Verificar Credenciales de Usuario

Consulta utilizada en el login para validar email y contraseña.

-- Verificar usuario por email y contraseña SELECT usuario_id, nombre, email, rol, activo FROM usuarios WHERE email = ? AND activo = 1; -- Luego verificar password con password_verify() en PHP -- password_verify($password_ingresado, $password_hash_bd)

2.2 Listar Todos los Usuarios del Sistema

Consulta para obtener todos los usuarios con su información básica.

SELECT usuario_id AS 'ID', nombre AS 'Nombre Completo', email AS 'Correo Electrónico', rol AS 'Rol', CASE WHEN activo = 1 THEN 'Activo' ELSE 'Inactivo' END AS 'Estado', DATE_FORMAT(fecha_creacion, '%d/%m/%Y %H:%i') AS 'Fecha Creación' FROM usuarios ORDER BY usuario_id;

2.3 Contar Usuarios por Rol

Estadística de usuarios agrupados por rol.

SELECT rol AS 'Rol', COUNT(*) AS 'Total', SUM(CASE WHEN activo = 1 THEN 1 ELSE 0 END) AS 'Activos', SUM(CASE WHEN activo = 0 THEN 1 ELSE 0 END) AS 'Inactivos' FROM usuarios GROUP BY rol ORDER BY COUNT(*) DESC;

3. Consultas de Asignaciones

3.1 Listar Asignaciones Completas con JOIN

Consulta principal para mostrar asignaciones con toda la información relacionada.

SELECT a.asignacion_id, f.codigo_ficha, f.programa, CONCAT(i.nombre, ' ', i.apellido) AS instructor, i.especialidad, e.nombre_experiencia AS competencia, amb.nombre_ambiente AS ambiente, a.fecha_inicio, a.fecha_fin, a.hora_inicio, a.hora_fin, a.dias_semana, a.estado, a.observaciones FROM asignaciones a INNER JOIN fichas f ON a.ficha_id = f.ficha_id INNER JOIN instructores i ON a.instructor_id = i.id INNER JOIN experiencias e ON a.experiencia_id = e.experiencia_id LEFT JOIN ambientes amb ON a.ambiente_id = amb.ambiente_id ORDER BY a.fecha_inicio DESC;

3.2 Asignaciones por Instructor

Consulta para ver todas las asignaciones de un instructor específico.

SELECT f.codigo_ficha, f.programa, e.nombre_experiencia, a.fecha_inicio, a.fecha_fin, a.dias_semana, a.estado FROM asignaciones a INNER JOIN fichas f ON a.ficha_id = f.ficha_id INNER JOIN experiencias e ON a.experiencia_id = e.experiencia_id WHERE a.instructor_id = ? AND a.estado IN ('Programada', 'En Curso') ORDER BY a.fecha_inicio;

3.3 Verificar Conflictos de Horario (Instructor)

Consulta para detectar si un instructor ya tiene asignación en el horario propuesto.

SELECT COUNT(*) as conflictos FROM asignaciones WHERE instructor_id = ? AND ( (? BETWEEN hora_inicio AND hora_fin) OR (? BETWEEN hora_inicio AND hora_fin) OR (hora_inicio BETWEEN ? AND ?) ) AND ( dias_semana LIKE CONCAT('%', ?, '%') OR ? LIKE CONCAT('%', dias_semana, '%') ) AND ? <= fecha_fin AND ? >= fecha_inicio;

3.4 Verificar Conflictos de Ambiente

Consulta para detectar si un ambiente ya está ocupado en el horario propuesto.

SELECT COUNT(*) as conflictos FROM asignaciones WHERE ambiente_id = ? AND ( (? BETWEEN hora_inicio AND hora_fin) OR (? BETWEEN hora_inicio AND hora_fin) OR (hora_inicio BETWEEN ? AND ?) ) AND ( dias_semana LIKE CONCAT('%', ?, '%') OR ? LIKE CONCAT('%', dias_semana, '%') ) AND ? <= fecha_fin AND ? >= fecha_inicio;

4. Consultas de Notificaciones

4.1 Listar Notificaciones del Instructor

Consulta para obtener notificaciones enviadas a un instructor.

SELECT id, tipo, titulo, mensaje, leida, DATE_FORMAT(fecha_creacion, '%d/%m/%Y %H:%i') as fecha FROM notificaciones_instructor WHERE instructor_id = ? ORDER BY fecha_creacion DESC LIMIT 20;

4.2 Contar Notificaciones No Leídas

Consulta para obtener el contador de notificaciones pendientes.

SELECT COUNT(*) as no_leidas FROM notificaciones_instructor WHERE instructor_id = ? AND leida = 0;

Consulta para obtener solicitudes de cambio de perfil de instructores.

SELECT id, instructor_id, instructor_nombre, tipo, titulo, mensaje, leida, DATE_FORMAT(fecha_creacion, '%d/%m/%Y %H:%i') as fecha FROM notificaciones_coordinador WHERE coordinador_id = ? AND tipo = 'cambio_perfil' ORDER BY fecha_creacion DESC;

4.4 Marcar Notificación como Leída

Consulta para actualizar el estado de una notificación.

UPDATE notificaciones_instructor SET leida = 1, fecha_lectura = NOW() WHERE id = ? AND instructor_id = ?;

Consulta para crear una nueva notificación.

INSERT INTO notificaciones_instructor (instructor_id, coordinador_id, tipo, titulo, mensaje, leida, fecha_creacion) VALUES (?, ?, ?, ?, ?, 0, NOW());

6. Consultas de Instructores

6.1 Listar Todos los Instructores

Consulta para obtener la lista completa de instructores.

SELECT id, CONCAT(nombre, ' ', apellido) AS nombre_completo, email, telefono, especialidad, registro, DATE_FORMAT(fecha_ingreso, '%d/%m/%Y') as fecha_ingreso FROM instructores ORDER BY apellido, nombre;

6.2 Buscar Instructor por Email

Consulta para buscar un instructor específico.

SELECT id, nombre, apellido, email, telefono, documento, especialidad FROM instructores WHERE email = ?;

6.3 Actualizar Perfil de Instructor

Consulta para actualizar información del instructor.

UPDATE instructores SET nombre = ?, apellido = ?, email = ?, telefono = ?, documento = ?, especialidad = ? WHERE id = ?;

6.4 Estadísticas de Instructores

Consulta para obtener estadísticas de carga de instructores.

SELECT i.id, CONCAT(i.nombre, ' ', i.apellido) AS instructor, i.especialidad, COUNT(a.asignacion_id) AS total_asignaciones, SUM(CASE WHEN a.estado = 'En Curso' THEN 1 ELSE 0 END) AS asignaciones_activas FROM instructores i LEFT JOIN asignaciones a ON i.id = a.instructor_id GROUP BY i.id, i.nombre, i.apellido, i.especialidad ORDER BY total_asignaciones DESC;

7. Consultas de Fichas y Programas

7.1 Listar Fichas Activas

Consulta para obtener fichas en estado activo.

SELECT ficha_id, codigo_ficha, programa, DATE_FORMAT(fecha_inicio, '%d/%m/%Y') as inicio, DATE_FORMAT(fecha_fin, '%d/%m/%Y') as fin, estado, s.nombre_sede FROM fichas f LEFT JOIN sedes s ON f.sede_id = s.sede_id WHERE estado = 'Activa' ORDER BY fecha_inicio DESC;

7.2 Fichas con Conteo de Aprendices

Consulta para ver fichas con cantidad de aprendices.

SELECT f.codigo_ficha, f.programa, COUNT(ap.aprendiz_id) AS total_aprendices, SUM(CASE WHEN ap.estado = 'Activo' THEN 1 ELSE 0 END) AS aprendices_activos FROM fichas f LEFT JOIN aprendices ap ON f.ficha_id = ap.ficha_id GROUP BY f.ficha_id, f.codigo_ficha, f.programa ORDER BY total_aprendices DESC;

7.3 Listar Programas de Formación

Consulta para obtener todos los programas disponibles.

SELECT programa_id, codigo, nombre, nivel, duracion_meses, estado FROM programas WHERE estado = 'Activo' ORDER BY nombre;

7.4 Fichas por Programa

Consulta para agrupar fichas por programa.

SELECT programa, COUNT(*) AS total_fichas, SUM(CASE WHEN estado = 'Activa' THEN 1 ELSE 0 END) AS fichas_activas FROM fichas GROUP BY programa ORDER BY total_fichas DESC;

8. Consultas de Ambientes y Sedes

8.1 Listar Ambientes Disponibles

Consulta para obtener ambientes disponibles.

SELECT a.ambiente_id, a.nombre_ambiente, a.capacidad, a.tipo, a.estado, s.nombre_sede FROM ambientes a INNER JOIN sedes s ON a.sede_id = s.sede_id WHERE a.estado = 'Disponible' ORDER BY s.nombre_sede, a.nombre_ambiente;

8.2 Ambientes por Tipo

Consulta para agrupar ambientes por tipo.

SELECT tipo, COUNT(*) AS total, SUM(CASE WHEN estado = 'Disponible' THEN 1 ELSE 0 END) AS disponibles, SUM(CASE WHEN estado = 'Ocupado' THEN 1 ELSE 0 END) AS ocupados FROM ambientes GROUP BY tipo ORDER BY total DESC;

8.3 Listar Sedes

Consulta para obtener todas las sedes.

SELECT sede_id, nombre_sede, direccion, telefono, ciudad, departamento FROM sedes ORDER BY ciudad, nombre_sede;

8.4 Ambientes con Uso

Consulta para ver ambientes con estadísticas de uso.

SELECT a.nombre_ambiente, a.capacidad, a.tipo, COUNT(asig.asignacion_id) AS veces_asignado FROM ambientes a LEFT JOIN asignaciones asig ON a.ambiente_id = asig.ambiente_id GROUP BY a.ambiente_id, a.nombre_ambiente, a.capacidad, a.tipo ORDER BY veces_asignado DESC;

9. Consultas de Reportes y Estadísticas

9.1 Reporte de Asignaciones por Período

Consulta para generar reporte de asignaciones en un rango de fechas.

SELECT DATE_FORMAT(a.fecha_inicio, '%Y-%m') AS periodo, COUNT(*) AS total_asignaciones, COUNT(DISTINCT a.instructor_id) AS instructores_activos, COUNT(DISTINCT a.ficha_id) AS fichas_atendidas FROM asignaciones a WHERE a.fecha_inicio BETWEEN ? AND ? GROUP BY DATE_FORMAT(a.fecha_inicio, '%Y-%m') ORDER BY periodo DESC;

9.2 Dashboard - Estadísticas Generales

Consulta para obtener estadísticas del dashboard.

SELECT (SELECT COUNT(*) FROM instructores) AS total_instructores, (SELECT COUNT(*) FROM fichas WHERE estado = 'Activa') AS fichas_activas, (SELECT COUNT(*) FROM asignaciones WHERE estado IN ('Programada', 'En Curso')) AS asignaciones_activas, (SELECT COUNT(*) FROM ambientes WHERE estado = 'Disponible') AS ambientes_disponibles, (SELECT COUNT(*) FROM aprendices WHERE estado = 'Activo') AS aprendices_activos;

9.3 Reporte de Carga Horaria por Instructor

Consulta para calcular horas asignadas por instructor.

SELECT CONCAT(i.nombre, ' ', i.apellido) AS instructor, i.especialidad, COUNT(a.asignacion_id) AS total_asignaciones, SUM(a.horas_semanales) AS horas_semanales_total, AVG(a.horas_semanales) AS promedio_horas FROM instructores i LEFT JOIN asignaciones a ON i.id = a.instructor_id WHERE a.estado IN ('Programada', 'En Curso') GROUP BY i.id, i.nombre, i.apellido, i.especialidad ORDER BY horas_semanales_total DESC;

9.4 Reporte de Ocupación de Ambientes

Consulta para ver ocupación de ambientes por día.

SELECT amb.nombre_ambiente, amb.tipo, COUNT(a.asignacion_id) AS asignaciones_activas, GROUP_CONCAT(DISTINCT a.dias_semana SEPARATOR ', ') AS dias_ocupados FROM ambientes amb LEFT JOIN asignaciones a ON amb.ambiente_id = a.ambiente_id AND a.estado IN ('Programada', 'En Curso') GROUP BY amb.ambiente_id, amb.nombre_ambiente, amb.tipo ORDER BY asignaciones_activas DESC;

9.5 Reporte de Aprendices por Ficha

Consulta para generar reporte de aprendices.

SELECT f.codigo_ficha, f.programa, COUNT(ap.aprendiz_id) AS total_aprendices, SUM(CASE WHEN ap.estado = 'Activo' THEN 1 ELSE 0 END) AS activos, SUM(CASE WHEN ap.estado = 'Retirado' THEN 1 ELSE 0 END) AS retirados, SUM(CASE WHEN ap.estado = 'Graduado' THEN 1 ELSE 0 END) AS graduados FROM fichas f LEFT JOIN aprendices ap ON f.ficha_id = ap.ficha_id GROUP BY f.ficha_id, f.codigo_ficha, f.programa ORDER BY total_aprendices DESC;

10. Consultas de Auditoría y Validación

10.1 Historial de Auditoría de Asignaciones

Consulta para ver el historial de cambios en asignaciones.

SELECT id_auditoria, asignacion_id, usuario_que_creo, DATE_FORMAT(fecha_registro, '%d/%m/%Y %H:%i:%s') as fecha, detalles FROM auditoria_asignaciones ORDER BY fecha_registro DESC LIMIT 50;

10.2 Verificar Integridad de Datos

Consulta para verificar la integridad referencial.

-- Verificar asignaciones sin instructor SELECT COUNT(*) as asignaciones_huerfanas FROM asignaciones a LEFT JOIN instructores i ON a.instructor_id = i.id WHERE i.id IS NULL; -- Verificar asignaciones sin ficha SELECT COUNT(*) as asignaciones_sin_ficha FROM asignaciones a LEFT JOIN fichas f ON a.ficha_id = f.ficha_id WHERE f.ficha_id IS NULL;

10.3 Validar Fechas de Asignaciones

Consulta para detectar asignaciones con fechas inválidas.

SELECT asignacion_id, fecha_inicio, fecha_fin, DATEDIFF(fecha_fin, fecha_inicio) as duracion_dias FROM asignaciones WHERE fecha_fin <= fecha_inicio OR fecha_inicio < CURDATE() ORDER BY fecha_inicio;

10.4 Detectar Sobrecargas de Instructores

Consulta para identificar instructores con más de 20 horas semanales.

SELECT i.id, CONCAT(i.nombre, ' ', i.apellido) AS instructor, SUM(a.horas_semanales) AS total_horas_semanales, COUNT(a.asignacion_id) AS total_asignaciones FROM instructores i INNER JOIN asignaciones a ON i.id = a.instructor_id WHERE a.estado IN ('Programada', 'En Curso') GROUP BY i.id, i.nombre, i.apellido HAVING total_horas_semanales > 20 ORDER BY total_horas_semanales DESC;
⚠️ Nota sobre Triggers:
El sistema tiene triggers automáticos que validan:
• Carga horaria máxima de 20 horas por asignación
• Conflictos de horario de instructores
• Conflictos de ocupación de ambientes
• Validación de fechas de fichas
• Auditoría automática de cambios

11. Consultas Avanzadas y Optimización

11.1 Vista de Asignaciones Completas

Crear vista para simplificar consultas complejas.

CREATE OR REPLACE VIEW vista_asignaciones_completas AS SELECT a.asignacion_id, a.fecha_inicio, a.fecha_fin, a.hora_inicio, a.hora_fin, a.dias_semana, a.estado, f.codigo_ficha, f.programa, CONCAT(i.nombre, ' ', i.apellido) AS instructor_nombre, i.email AS instructor_email, i.especialidad, e.nombre_experiencia, e.duracion_horas, amb.nombre_ambiente, amb.tipo AS tipo_ambiente, s.nombre_sede FROM asignaciones a INNER JOIN fichas f ON a.ficha_id = f.ficha_id INNER JOIN instructores i ON a.instructor_id = i.id INNER JOIN experiencias e ON a.experiencia_id = e.experiencia_id LEFT JOIN ambientes amb ON a.ambiente_id = amb.ambiente_id LEFT JOIN sedes s ON f.sede_id = s.sede_id;

11.2 Usar la Vista Creada

Consulta simplificada usando la vista.

-- Consulta simple usando la vista SELECT * FROM vista_asignaciones_completas WHERE estado = 'En Curso' ORDER BY fecha_inicio DESC; -- Filtrar por instructor SELECT * FROM vista_asignaciones_completas WHERE instructor_email = 'josevera@gmail.com'; -- Filtrar por sede SELECT * FROM vista_asignaciones_completas WHERE nombre_sede = 'Sede Pescadero';

11.3 Búsqueda Full-Text

Consulta para búsqueda avanzada en múltiples campos.

SELECT asignacion_id, codigo_ficha, programa, instructor_nombre, nombre_experiencia FROM vista_asignaciones_completas WHERE CONCAT(codigo_ficha, ' ', programa, ' ', instructor_nombre, ' ', nombre_experiencia) LIKE CONCAT('%', ?, '%') ORDER BY fecha_inicio DESC LIMIT 20;

11.4 Consulta con Paginación

Consulta optimizada para mostrar resultados paginados.

-- Página 1 (primeros 10 registros) SELECT SQL_CALC_FOUND_ROWS * FROM vista_asignaciones_completas ORDER BY fecha_inicio DESC LIMIT 0, 10; -- Obtener total de registros SELECT FOUND_ROWS() as total_registros;
💡 Tip de Optimización:
Las vistas simplifican consultas complejas y mejoran la mantenibilidad del código. Use índices en columnas frecuentemente consultadas (email, fecha_inicio, estado).

12. Consultas de Mantenimiento y Limpieza

12.1 Limpiar Notificaciones Antiguas

Consulta para eliminar notificaciones leídas de más de 30 días.

-- Eliminar notificaciones de instructores antiguas DELETE FROM notificaciones_instructor WHERE leida = 1 AND fecha_lectura < DATE_SUB(NOW(), INTERVAL 30 DAY); -- Eliminar notificaciones de coordinador antiguas DELETE FROM notificaciones_coordinador WHERE leida = 1 AND fecha_creacion < DATE_SUB(NOW(), INTERVAL 30 DAY);

12.2 Actualizar Estado de Asignaciones

Consulta para actualizar automáticamente asignaciones finalizadas.

UPDATE asignaciones SET estado = 'Finalizada' WHERE fecha_fin < CURDATE() AND estado IN ('Programada', 'En Curso');

12.3 Actualizar Estado de Fichas

Consulta para actualizar fichas que ya finalizaron.

UPDATE fichas SET estado = 'Finalizada' WHERE fecha_fin < CURDATE() AND estado = 'Activa';

12.4 Limpiar Eventos Antiguos del Calendario

Consulta para eliminar eventos de más de 6 meses.

DELETE FROM events WHERE end_date < DATE_SUB(NOW(), INTERVAL 6 MONTH);

12.5 Optimizar Tablas

Comandos para optimizar el rendimiento de la base de datos.

-- Optimizar tabla de asignaciones OPTIMIZE TABLE asignaciones; -- Optimizar tabla de notificaciones OPTIMIZE TABLE notificaciones_instructor; OPTIMIZE TABLE notificaciones_coordinador; -- Analizar tabla para actualizar estadísticas ANALYZE TABLE asignaciones;
⚠️ Precaución:
Ejecute estas consultas de mantenimiento en horarios de baja actividad. Haga backup de la base de datos antes de ejecutar operaciones de limpieza masiva.

15. Consultas de Seguridad y Control de Acceso

15.1 Auditar Intentos de Login

Consulta para registrar intentos de acceso.

-- Crear tabla de auditoría de login (si no existe) CREATE TABLE IF NOT EXISTS auditoria_login ( id INT AUTO_INCREMENT PRIMARY KEY, email VARCHAR(100), ip_address VARCHAR(45), user_agent TEXT, exitoso TINYINT(1), fecha_intento TIMESTAMP DEFAULT CURRENT_TIMESTAMP, INDEX idx_email (email), INDEX idx_fecha (fecha_intento) ); -- Registrar intento de login INSERT INTO auditoria_login (email, ip_address, user_agent, exitoso) VALUES (?, ?, ?, ?);

15.2 Detectar Intentos Fallidos Múltiples

Consulta para identificar posibles ataques de fuerza bruta.

SELECT email, COUNT(*) as intentos_fallidos, MAX(fecha_intento) as ultimo_intento FROM auditoria_login WHERE exitoso = 0 AND fecha_intento > DATE_SUB(NOW(), INTERVAL 1 HOUR) GROUP BY email HAVING intentos_fallidos >= 5 ORDER BY intentos_fallidos DESC;

15.3 Actualizar Último Acceso

Consulta para registrar el último acceso del usuario.

UPDATE usuarios SET ultimo_acceso = NOW() WHERE usuario_id = ?;

15.4 Desactivar Usuario

Consulta para desactivar temporalmente un usuario.

UPDATE usuarios SET activo = 0 WHERE usuario_id = ?; -- Reactivar usuario UPDATE usuarios SET activo = 1 WHERE usuario_id = ?;
🔒 Seguridad:
• Las contraseñas se almacenan hasheadas con bcrypt
• Use prepared statements para prevenir SQL injection
• Implemente rate limiting en intentos de login
• Registre todos los accesos al sistema
• Revise logs de auditoría regularmente
